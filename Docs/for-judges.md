# Don't Catch That Bird — 评委手册

> 金海豚 GameJam 2026 · 单人 2 天开发 · Unity 2022

---

## 一句话简介

在 37 键虚拟键盘上，抓住跳跃的小鸟，避开危险键——用反应力挑战自己的极限。

---

## 核心玩法

游戏开始后，一只手绘小鸟随机停留在键盘的某个按键上，每隔 0.6~2.5 秒自动跳到新键。右上角显示当前危险键（DangerKey）。玩家必须：

- 按鸟所在的键 → 抓取成功，得分
- 按 DangerKey → 立即 Game Over，全屏红闪
- 按其他键 → Combo 清零，Miss +1
- Miss 达到 5 → Game Over

### 计分公式

```
每次得分 = 10 + 当前连击 × 5
```

Combo 越高单次得分越高，但按错一次全部归零。

### 难度曲线设计

游戏设计了 4 档难度区间，随分数自动升级：

| 分数 | 鸟跳跃间隔 | DangerKey 数量 | 设计意图 |
|---|---|---|---|
| 0~49 | 2.0~2.5s | 1 个 | 新手适应期，宽松节奏 |
| 50~99 | 1.5~2.0s | 1 个 | 节奏加快 |
| 100~199 | 1.0~1.5s | 2 个 | **注意力分配考验** — 需要同时关注两个危险目标 |
| 200+ | 0.6~1.0s | 3 个 | 极限反应阶段 |

100 分是游戏的转折点——DangerKey 从 1 个变成 2 个。这不仅是数值变化，而是对玩家认知负荷的挑战。玩家必须从"追踪一个目标"切换到"同时监控多个目标"的信息处理模式。

### 陷阱机制

小鸟可能跳到 DangerKey 上。此时该键既是鸟键又是危险键：

- 跳上去的瞬间**不计 Miss**
- 但如果玩家按下去 → **即时 Game Over**

这制造了一种特殊的心理张力：眼睁睁看着鸟坐在死亡键上，你只能等待它自己跳走。

## 道具系统

每抓取 8~12 只小鸟（道具持续期间的抓取不计入计数）出现一个道具：

| 道具 | 效果 | 设计意图 |
|---|---|---|
| 冻结（雪花） | 鸟静止 5 秒 | 打断紧张节奏，给玩家喘息机会 |
| 无敌（心形） | 4 秒内不扣 Combo/Miss，DangerKey 不死 | 鼓励玩家尝试高风险操作 |

道具持续时间随分数递减（2.5s → 1s），避免道具在后期过于强势。

---

## 金鸟系统

Combo 达到 30 的倍数时，下一只鸟变为金色。抓到金鸟得分 ×2，并爆出 20 个金色粒子。

设计意图：为连击提供额外的奖励节点，激励玩家维持高 Combo。

---

## 视觉与反馈

游戏采用黑白手绘简笔画风格，所有 UI 元素带有粗黑描边和 Perlin Noise 抖动：

- **按键三色反馈**：抓取闪绿、错误闪红、危险键深红 + 全屏红闪
- **Combo 颜色升级**：蓝色 → 橙色 → 红色，字号递增
- **抓取粒子**：彩色方块爆出 + 分数数字飘浮淡出
- **道具遮罩**：冻结淡蓝、无敌淡粉全屏覆盖
- **字体**：STHUPO 繁圆手绘字体替代 GPU Shader 抖动方案

## 技术实现要点

| 项目 | 说明 |
|---|---|
| 架构 | GameManager 单例 + 回调注入（Bird 不直接依赖 GameManager） |
| 数据驱动 | 所有参数通过 GameConfigSO ScriptableObject 暴露，无需改代码即可调参 |
| 代码质量 | 经过 14 轮重构，所有已知空引用路径已防御 |
| 解耦 | 粒子效果、音效、道具、DangerKey 选择均独立为模块 |
| 降级策略 | 无美术资源时自动回退到几何体占位，无音效时正弦波 beep 占位 |

## 操作一览

| 操作 | 功能 |
|---|---|
| 按鸟键 | 抓取 |
| 按 DangerKey | Game Over |
| 按其他键 | Combo 清零 + Miss |
| 空格 | 开始 / 重开 |
| Tab | 帮助面板 |
| Esc | 退出 |
| Alt | 显示鼠标 |
